# Style Guide

## Brand feeling
The site should feel:
- alive
- playful
- bold
- internet-native
- sticky
- colorful without becoming chaotic

## Design model
This system uses color as a **mood engine**, not as decoration.

Every screen should feel like:
- a scene
- a brand world
- a character-driven experience
- a clear emotional tone

## Palette structure

### Core colors
- Sky Base: `#77C7F5`
- Electric Indigo: `#4D46F4`
- Hot Pink: `#FF4F9A`
- Soft White: `#F8F7F4`

### Secondary colors
- Lavender Pink: `#F2A8E8`
- Deep Blue-Violet: `#4B45D8`
- Candy Purple: `#7B61FF`

### Illustration-only accents
- Lemon Yellow: `#F6DD4E`
- Mint Green: `#6BE38C`
- Bright Cyan: `#56C8FF`

## Palette rules
- use 3 to 5 dominant colors total
- use 1 dominant background color per section
- do not make every headline multicolor
- do not use accent color on large paragraphs
- do not introduce gray as a main mood color
- keep utility neutrals minimal

## Section formula
Each section should define:
- dominant background color
- primary text color
- accent word color
- CTA color
- optional decorative color

## CTA system

### Primary CTA
- background: electric indigo
- text: soft white
- shape: rounded, chunky, obvious

### Secondary CTA
- background: soft white
- text: electric indigo
- shape: pill or rounded rectangle

### Playful CTA
- background: hot pink
- text: soft white
- reserved for social, generators, or high-energy moments

## Typography behavior
- headlines should be oversized and punchy
- one phrase can carry accent color emphasis
- supporting copy should stay simpler than the headline
- avoid too many font styles at once

## Background behavior
- large flat color fields
- soft gradients only if they do not muddy the scene
- avoid noisy backgrounds behind important text

## Component rules
- cards should support the section, not compete with it
- nav buttons should feel like branded objects, not utility defaults
- icons should echo the same color family as the UI
- image treatments should feel part of the same world as the interface

## Anti-patterns
Avoid:
- gray-heavy SaaS palettes
- muddy gradients
- washed-out pinks and blues
- too many competing accents
- low-contrast CTA buttons
- character art that clashes with the interface colors
