# Build Workflow

## Goal
Use this workflow to replicate the energy of colorful internet-native websites without copying them directly.

## Step 1: Collect references
Gather screenshots for:
- hero
- navigation
- CTA moments
- social proof
- feature sections
- footer

Put them in `reference-screenshots/`.

## Step 2: Analyze the references
For each screenshot, answer:
- what is the dominant background color?
- what is the main text color?
- what is the accent color?
- what makes the CTA obvious?
- what emotional tone does the scene create?

## Step 3: Define your brand translation
Do not copy the exact hues unless that is intentional.
Translate the system into your own brand.

Example:
- keep the sky-blue openness
- swap hot pink for coral if needed
- keep indigo for primary button structure

## Step 4: Build the color map
Create a section plan like:
- Hero: sky blue + indigo + white
- Community: indigo + white + pink
- Tool feature: pastel pink + white + indigo
- Footer CTA: deep violet + white + hot pink

## Step 5: Build the component library
Standardize:
- nav pills
- primary CTA
- secondary CTA
- card styles
- headline patterns
- icon treatment

## Step 6: Test for dead zones
Look for sections where:
- all colors flatten into one tone
- the CTA disappears
- too many accents compete
- the page loses emotional rhythm

## Step 7: Refine with AI
Use the prompts in `prompts/` to:
- rebalance contrast
- increase color confidence
- tighten hierarchy
- generate alternate section treatments

## Step 8: Document the final system
Update the repo after each improvement so the repo stays your actual source of truth.
